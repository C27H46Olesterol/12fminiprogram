# 售后服务平台需求规格说明书 (Requirements Specification)

本通过定义各角色功能与业务流程，指导售后服务平台的开发与实施。

## 1. 用户角色定义 (User Roles)

系统共包含 8 类用户角色，每类角色拥有特定的权限与功能。

### 1.1 普通用户 (General User)
- **角色描述**：未注册或未认证的访客用户。
- **核心功能**：
  - **信息浏览**：浏览产品参数、查看企业新闻、浏览新品广告轮播图。
  - **身份转化**：
    - **激活产品**：输入产品信息激活，转化为“产品用户”。
    - **注册维修工**：提交店铺及个人信息，申请成为“注册维修工”。

### 1.2 产品用户 (Product User)
- **角色描述**：拥有已激活产品的终端消费者。
- **核心功能**：
  - **我的产品**：查看已激活产品的详细信息、质保期限。
  - **售后服务**：
    - **在线报修**：提交报修申请，生成维修工单（包含故障描述、图片等）。
    - **人工服务**：一键拨打售后客服电话。
  - **进度查询**：查看工单处理进度（待接单、维修中、已完成）。

### 1.3 注册维修工 (Registered Repairman)
- **角色描述**：经平台认证的专业维修人员，可自主接单或接受指派。
- **核心功能**：
  - **工单管理**：
    - **自主抢单**：基于地理位置（LBS）接收并认领附近的报修申请。
    - **指派接单**：接收后台服务人员或区域经销商指派的工单。
  - **维修流程**：
    - **联系用户**：查看工单详情并联系用户确认故障。
    - **配件申请**：向区域经销商申请维修所需配件或整机。
    - **完工反馈**：提交维修记录、上传修复照片，提交审核。
  - **结算管理**：查看工时费到账情况及历史收入。

### 1.4 后台服务人员 (Back-office Support)
- **角色描述**：平台内部客服或调度人员。
- **核心功能**：
  - **工单调度**：收到用户报修申请后，主动联系用户初筛问题。
  - **派单管理**：查询附近的注册维修工，进行人工指派。

### 1.5 区域经销商 (Regional Distributor)
- **角色描述**：特定区域的维修工管理者及物资供应方。
- **核心功能**：
  - **人员管理**：管理管辖区域内的注册维修工。
  - **配件/库存管理**：
    - 处理维修工的配件/整机申请（用于售后或售卖）。
    - 管理自身仓库库存。

### 1.6 区域服务员 (Regional Service Staff)
- **角色描述**：协助区域经销商运营的服务人员。
- **核心功能**：
  - **信息查看**：查看所服务区域经销商的经营数据、库存信息及下属维修工状态。

### 1.7 平台管理员 (Platform Administrator)
- **角色描述**：负责平台内容运营。
- **核心功能**：
  - **内容发布**：管理企业新闻、产品库信息。
  - **广告管理**：配置首页轮播图、新品广告。

### 1.8 超级管理员 (Super Administrator)
- **角色描述**：系统最高权限拥有者。
- **核心功能**：
  - **全权管理**：拥有上述所有角色的管理权限。
  - **系统配置**：账号管理、权限分配、系统参数设置。

## 2. 核心业务流程 (Core Business Processes)

### 2.1 维修闭环流程
1.  **发起**：产品用户提交报修申请。
2.  **调度**：
    *   *自动/抢单*：附近维修工收到通知并抢单。
    *   *人工派单*：后台服务人员介入，联系用户确认后指派维修工。
3.  **维修**：
    *   维修工联系用户上门/远程处理。
    *   如需配件，维修工向区域经销商申请 -> 经销商发货 -> 维修工收货。
4.  **完成**：维修工提交完工证明 -> 后台/系统审核。
5.  **结算**：审核通过后，工时费自动结算至维修工账户。

### 2.2 身份认证流程
- **普通用户 -> 产品用户**：输入序列号/扫码 -> 系统校验 -> 绑定成功。
- **普通用户 -> 注册维修工**：提交资料（店铺照、资质） -> 后台/经销商审核 -> 开通权限。

## 3. 数据实体 (Key Data Entities)
- **用户 (User)**：基础信息、角色标签。
- **产品 (Product)**：型号、参数、详情。
- **设备实例 (Device)**：序列号、激活状态、归属用户、质保期。
- **工单 (Ticket)**：单号、状态、故障描述、关联设备、处理人、时间轴。
- **配件/库存 (Part/Inventory)**：SKU、数量、归属仓库。
